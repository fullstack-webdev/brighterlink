Analyze: App Overview

Analyze allows Team Member(s) to create dashboards.

Interface

Analyze is a Single Page Web Application within the BrighterLink platform. The Analyze interface is composed of the following parts:
  • Navigation Bar
  • Company Panel
  • Dashboard Panel
  • Control Bar
  • Date Range
  • Widgets

Navigation Bar

The Navigation Bar is the bar at the top of the interface that allows Team Members to manage their Profile and to select an App. It is visible on all BrighterLink Apps.

Company Panel

The Company Panel is on the right side of the interface that allows Team Members to manage their Account, other Team Members, Assets (ie. images, client logos), Sources (ie. data loggers, utility bills), and Groups. It is visible on all BrighterLink Apps.

Dashboard Panel

The Dashboard Panel is on the left side of the interface. It allows Team Members to manage their dashboards. It is only visible in Analyze (similar to the Presentations Panel in Present).

Control Bar

The Control Bar is the bar near the top of the interface that allows Team Member to manage the current dashboard. It is only visible in Analyze.

Date Range

The Date Range is the control near the top-right of the interface that allows Team Members to specify the date range for all widgets in the current dashboard.

Widgets

Widgets are the individual information items that fill a dashboard. They are viewable by Team Members and can be shared with others. Though serving a similar purpose, dashboards widgets are unrelated to Present widgets, which means that you cannot use a dashboard widget in a Present presentation.

Sources & Groups

Sources and Groups are part of the Control Bar and define the limit of data that can appear in the dashboard's widgets. They also define how to segment that data.

Sources

Sources are derived from the Company Panel. There are four levels of Source hierarchy:
  • Facility
    • Scope
      • Node
        • Metric
Facility
A Facility is the physical building to which a data source is related, such as where the hardware device called a "data logger" is installed.
Scope
A Scope is the data source related to a Facility that houses data from Nodesand sends that data to BrighterLink. A Scope, such as a SMA WebBox and eGauge 3000, uses various methods to send data to BrighterLink, including API, FTP Push, and HTTP. Any number of Scopescan be related to a single facility.
Facility
Node A Noder is a collection of measured data, such as a sensorconnected to an HVAC unit, solar array, elevator, etc. that includes various Metrics such as kW, kWh, water pressure, ambient lighting level, etc. Nodestransmit their measured data to Scopes, meaning that they do not require a direct Internet connection. Multiple Nodes, such as multiple SMA inverters, typically use an on-premise hub (e.g. the SMA WebBox) to collect the data that they measure.
Metric
Metrics are the measurements made by the Node such as kW, kWh, water pressure, ambient lighting level, etc.
Segment
A Segment is a grouping of data defined by one or more data sources.

Widget Editor

Team Members can edit widgets via a type-specific editor. Available types include:
  • Timeline - metrics across a time series
  • Bar - metrics comparison with horizontal (columnar) bars
  • Pie - metrics comparison with whole circular chart
  • Table - tabular view of metrics
  • Image
  • Equivalencies - metrics represented in real-world analogues
Additional types on development path:
  • Treemap - hierarchical nested rectangle (http://goo.gl/FdopUu)
  • Bullet - compact metrics comparison
  • Gauge - single value highlight
  • Radar - spider chart
  • Geo - quantitative metrics on a geographical map
  • Bubble - visualize metrics across two to four dimensions (http://goo.gl/YbxEcY)
  • Funnel - represents stages in a process
  • KPI - single number for headline
  • Relational - relationship analysis
  • Calendar - metrics over the course of a long span of time (http://goo.gl/rNgyWg)
  • andlestick - opening and closing metrics in a period overlaid on top of a total variance (http://goo.gl/Fi9ygh)
  • Histogram - grouped metrics across segments (http://goo.gl/06puYn)
  • Sankey - metric flow from one set of values to another (http://goo.gl/8cywg0)
  • Scatter - single datapoints plotted on a graph (http://goo.gl/UuTQJn)
  • Trendlines - straight line that most closely approximates the plotted data (http://goo.gl/oy8HbQ)
All data-based widgets (i.e. widgets that use data from sources) utilize two elements to define their structure:
  • Dashboard Metrics
  • Dashboard Dimensions

Dashboard Metrics

When creating a dashboard widget, Metrics are values that define what type of data the widget should handle such as kWh, kW, water pressure, ambient lighting, etc.

Examples

Timeline

  • 1. A Team Member clicks “Add Dashboard” and selects "Blank Dashboard" as the type.
  • 2. A dashboard with no widgets opens in the interface with a Dashboard Panel on the left, Control Bar near the top (below the Navigation Bar), and Date Range at the top-right.
  • 3. The Team Member specifies "July 1-31" as the Date Range in the Dashboard.
  • 4. The Team Member specifies "FacilityA" as the Source. FaciltyA has 100 Sensors with various Metrics per Sensor, including the kWh metric.
  • 5. The Team Member clicks "Add a Widget" and selects "Timeline" as the type.
  • 6. The Team Member specifies "kWh" as the Metric and "kW" as the "Compare with" Metric.
  • 7. When the Team Member clicks "Apply,” the widget appears within the dashboard showing a timeline from July 1st to July 31st with kWh and kW as separate lines. The kWh and kW values are totals of the values from all 100 sensors on FaciltyA.

Examples with Dimensions

Count of Datapoints
Widget Type = Bar
Metric = kWh
Group Dimension = Count of Datapoints
Results: ONE bar with total number of datapoints in the segment. If there are two segments, would show two bars.

Widget Type = Bar
Metric = kWh
Group Dimension = Date
Pivot Dimension = Count of Datapoints
Reults: ONE bar for EACH DATE with total number of datapoints for that date.
Interval: time between datapoints
Widget Type = Bar
Basis = There are 14 minutes between datapointA and datapointB, 10 minutes between datapointB and datapointC, and 10 miniutes between datapointC and datapointD.
Metric = kWh
Group Dimension = Interval
Reults: One bar for each interval. So, there would be two bars with the second bar twice as tall as the first (because there are two intervals of 10 minutes and only one of 14).
Country: national origin of data source.
Widget Type = Bar Basis = MetricA (kWh) is in SensorA. SensorA has a latitude of 39 and longitude of 94 (i.e. Kansas City). MetricB (kWh) is in SensorB. SensorB has no latitude/longitude recorded. SensorB is in DataLoggerA. DataLoggerA has a latitude of 60 and a longitude of 30 (i.e. St. Petersburg, Russia).
Metric = kWh
Group Dimension = Country
Results: Two bars of equal height, one for United States, and one for Russia.

Note: The same concept for Country applies to State, City, and Zip Code.
